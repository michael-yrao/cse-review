Every rep is a **coded** rep by default, warmups included. Do not schedule 🟡 Shaky or
🟢 Clean reviews as "no-code blueprint" — write them as code from a blank page. A no-code
blueprint is now an explicit opt-in the learner asks for (time-boxed day), not the house
default for reviews.

**Why:** stated Jul 14, 2026. Blueprints kept surfacing the same failure mode — the
approach was recalled fine but the *pointer/boundary arithmetic* collapsed at the keyboard
(206, 424, 75, 567, 901 all failed there, never on the approach). A blueprint can't catch
what only breaks when you type it.

**How to apply:**
- Scaffold a file/attempt banner for **every** scheduled problem, not just active blocks
  and 🔴 retries. See [[feedback_coding_for_clean]].
- Warmup slots are still 15 min — two coded warmups per slot is tight. If the day would
  blow past the cap, trim warmup *count* (≤4/day) rather than downgrading a rep to no-code.
- The 🟢 Clean backlog burns down slower under this rule. Factor that into the 🟢-backlog
  policy call rather than reverting to blueprints.
