For a problem tracked under multiple methods (e.g. 42 Array vs Two-Pointer, 75 Dutch Flag vs Bucket Sort), pull the harder/alternate variant into active rotation **when the base method retires (🏆, streak 3)** — not while the base is still churning through 🟡/🟢 reps.

**Why:** the user wants the space/technique-optimized variant as a deliberate "level-up" *after* the primary method is truly locked in, not as extra parallel load while the base is still shaky. Adding it early just inflates daily count on a problem that isn't yet solid.

**How to apply:**
- Park the variant in the knowledge-expansion queue with an explicit trigger note: "pull into rotation when <base method> retires (🏆)."
- When logging a result that takes the base method to 🏆 (streak 3), check for a parked variant and promote it: create its review row and schedule it as a new problem.
- First concrete instance: [[project_overview]] — 42 Trapping Rain Water Two-Pointer waits on 42 Array reaching 🏆 (Array hit 🟢 streak 1 on Jul 8, 2026).
