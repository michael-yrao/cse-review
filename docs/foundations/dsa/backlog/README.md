# Application Backlog Pools — pull, not push

These are **pull pools**, not checklists. You *pull* problems from them gated by
patterns you've already learned — never march a list top-to-bottom. A 🟢 confirms
transfer; a 🟡/🔴 is a **diagnostic** pointing at a pattern to refresh, not a cue
to learn something ad-hoc. See the "Application — *pull, not push*" thread in
[`../study_guide.md`](../study_guide.md).

There are two pools, one per phase of the reach-beyond curriculum:

| Pool | When you pull from it | Gated by |
|------|-----------------------|----------|
| **Interview-sourced** | *during* the knowledge-expansion phase (post-NC150, Tier 1) | NC150 + Tier 1 techniques |
| [**Competitive-style**](competitive_style.md) | *after* knowledge expansion (Tier 2 / competitive) | Tier 1 + Tier 2 techniques |

**Interview-sourced pool:** for now this is the live company-frequency sources
named in `study_guide.md` (LeetCode company filter, NeetCode.io company lists,
Blind/Discuss) — kept external because company banks rotate. Treat the static
2021–2022 frequency PDFs as *pattern-distribution* signal only, not a checklist.

**Competitive-style pool:** the curated list in [`competitive_style.md`](competitive_style.md).
This is the symmetric second pool — it keeps the pull-not-push mechanic running
*past* the Interview-ROI line, once you've crossed into Tier 2 for competitive
depth.

## How to log a pull

A pulled problem enters `dsa_progress.md` exactly like any other — 🟢/🟡/🔴,
spaced-repetition dates and all. The pool tells you *what* to pull; the tracker
owns *when it comes back*.
